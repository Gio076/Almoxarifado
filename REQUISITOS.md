# LEVANTAMENTOS DE REQUISITOS

Arquivo de documentação muito comum em projetos de desenvolvimento de software

## Objetivo:
A função principal dele é centralizar e organizar o que o sistema deve fazer e quais condições ela deve atender.

# Requisitos funcionais:
Requisitos funcionais detalham o que o sistema deve fazer, ou seja, as operações, comportamentos e funcionalidades que o software precisa oferecer para cumprir seu propósito e atender às expectativas do usuário e do negócio. Eles são essenciais para guiar o desenvolvimento, testes e validação do sistema, garantindo que cada recurso funcione conforme planejado.

## Exemplos:
Sistemas de e-commerce: permitir que usuários pesquisem produtos, adicionem itens ao carrinho e finalizem compras.

Aplicativos bancários: possibilitar consulta de saldo, transferências e pagamento de contas.

Sistemas corporativos: gerar relatórios financeiros, controlar estoques e gerenciar funcionários.

Redes sociais: criar perfis, editar informações e interagir com outros usuários.

Softwares de análise de dados: buscar, filtrar e visualizar conjuntos de dados específicos.

# REQUISITOS NÃO FUNCIONAIS:
Os requisitos não funcionais (RNFs) são especificações que determinam como o software deve operar, em vez de definir o que ele faz. Diferentemente dos requisitos funcionais, que descrevem ações ou regras de negócio, os RNFs estão relacionados à qualidade do sistema, como desempenho, confiabilidade, escalabilidade, segurança e usabilidade.
